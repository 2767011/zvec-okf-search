# Многофакторная аутентификация

Для административных учётных записей требуется второй фактор. Резервные коды хранятся отдельно от основного пароля.

# Ротация секретов

Токены интеграций и пароли сервисных пользователей меняются по расписанию. Старые значения отзываются после проверки новых реквизитов.

# Реагирование на инциденты

При подозрительной активности дежурный фиксирует время, источник и затронутые системы. Затем изолирует узел и сохраняет журналы для расследования.

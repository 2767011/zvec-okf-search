# Контроль качества поиска

Файл `queries.json` содержит контрольные запросы и фрагменты, которые должны
присутствовать в ожидаемом результате.

Запуск против работающего сервиса:

```bash
okf-zvec benchmark \
  --file benchmarks/queries.json \
  --service-url http://127.0.0.1:8765 \
  --model e5 \
  --modes semantic,fts,hybrid
```

## Базовый результат

Дата: 2026-07-09. Корпус: 158 фрагментов OKF. Модель:
`intfloat/multilingual-e5-small`. Порог релевантности: `0.25`. Веса
семантики и FTS: `1:1`.

| Режим | Top-1 | Top-3 | MRR | Пропуски | Среднее время |
| --- | ---: | ---: | ---: | ---: | ---: |
| `semantic` | 70% | 100% | 0.83 | 0 | 733 мс |
| `fts` | 90% | 90% | 0.90 | 1 | 1 мс |
| `hybrid` | 100% | 100% | 1.00 | 0 | 533 мс |

FTS ожидаемо пропускает смысловой запрос `каталог товаров контрагента`, в
котором нет терминов целевого документа. Гибридный режим компенсирует этот
случай семантической веткой.

Benchmark отправляет `no_cache=1`, поэтому задержка отражает выполнение поиска,
а не повторную выдачу из серверного кэша.

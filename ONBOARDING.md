# Онбординг

## 1. Как движутся данные

1. Клиент загружает каталог OKF Markdown через `POST /sync`.
2. Сервис разбивает файлы на фрагменты по заголовкам и элементам списков.
3. Для каждого фрагмента строятся эмбеддинги обеими настроенными моделями.
4. Русский текст лемматизируется и отдельно индексируется для BM25.
5. Поиск работает в семантическом, полнотекстовом или гибридном режиме с RRF.

Загруженный Markdown остаётся источником истины. Индексы zvec являются
производными данными, и их всегда можно перестроить.

## 2. Подготовка сервера

Используйте Debian или Ubuntu с оперативной памятью не менее 8 ГБ. Проверьте
окружение:

```bash
python3 --version
free -h
df -h /opt
```

Если памяти меньше 8 ГБ, включите swap до загрузки и запуска обеих моделей
эмбеддингов.

## 3. Установка

```bash
sudo apt-get update
sudo apt-get install -y git python3 python3-venv
git clone https://github.com/2767011/zvec-okf-search.git
cd okf-zvec-search
sudo ./deploy/install.sh
```

Установщик:

- создаёт `/opt/okf-zvec-search`;
- создаёт виртуальное окружение Python;
- устанавливает CPU-версию PyTorch и зависимости проекта;
- заранее загружает обе модели эмбеддингов;
- генерирует токен синхронизации;
- устанавливает службу systemd.

Установщик выводит токен один раз. Сохраните его в локальном файле, который
исключён из Git.

## 4. Запуск и проверка

```bash
sudo systemctl enable --now okf-zvec-search
sudo systemctl status okf-zvec-search
curl http://127.0.0.1:8765/health
```

Первый запуск может занять несколько десятков секунд, пока обе модели
загружаются в память.

## 5. Загрузка базы знаний

Windows:

```powershell
.\scripts\sync.ps1 `
  -OkfPath C:\путь\к\okf `
  -ServiceUrl http://АДРЕС_СЕРВЕРА:8765 `
  -TokenFile C:\защищённый-каталог\okf-zvec-token
```

Linux:

```bash
tar -C /путь/к -czf /tmp/okf.tgz okf
curl --fail \
  -H "X-OKF-Zvec-Token: $(cat /защищённый-каталог/okf-zvec-token)" \
  -H "Content-Type: application/gzip" \
  --data-binary @/tmp/okf.tgz \
  http://АДРЕС_СЕРВЕРА:8765/sync
```

## 6. Сравнение режимов

```powershell
.\scripts\search.ps1 "кассами" -Mode semantic
.\scripts\search.ps1 "кассами" -Mode fts
.\scripts\search.ps1 "кассами" -Mode hybrid
```

Используйте `fts` для быстрых запросов с точными терминами, `semantic` для
поиска по смыслу и `hybrid` как основной режим.

## 7. Повседневная эксплуатация

Просмотр журнала:

```bash
journalctl -u okf-zvec-search -f
```

Перезапуск:

```bash
sudo systemctl restart okf-zvec-search
```

Обновление:

```bash
git pull
sudo ./deploy/install.sh
sudo systemctl restart okf-zvec-search
```

## Диагностика

### Сервис долго запускается

До начала прослушивания HTTP сервис загружает в память обе модели. Проверьте
`journalctl`, свободную оперативную память и swap.

### FTS находит английский текст, но не русский

Индекс должен использовать токенизатор `whitespace` поверх лемматизированного
текста. После изменения токенизатора перестройте индекс через `POST /sync`.

### Синхронизация отвечает HTTP 401

Токен клиента не совпадает с токеном в
`${OKF_ZVEC_HOME}/config/service-token`.

### Hugging Face предупреждает об ограничении запросов

Укажите `HF_TOKEN` в `/etc/okf-zvec-search.env`, затем повторно запустите
установщик или перезапустите загрузку моделей. Не добавляйте этот файл в Git.

### Старые индексы занимают место

При успешной синхронизации создаются неизменяемые каталоги индексов с метками
времени. Это позволяет не заменять файлы, которые читаются активным процессом.
Остановите сервис на техническое обслуживание и удалите неактивные каталоги.
